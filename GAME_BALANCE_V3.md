## 2. 핵심 시스템 공식

### 2.1 스탯 소프트 캡 (Soft Cap)
모든 계산에 앞서 스탯 300 초과분에 대해 완만한 감쇄를 적용합니다.
- **공식:** `getEffectiveStat300(Stat)`
    - $Stat \le 300$: $Stat$
    - $Stat > 300$: $300 + (Stat - 300) \times 0.2$
    - *예시: 스탯 600 달성 시 유효 스탯은 360으로 계산됨*

### 2.2 전투 능력치 산출 (Derived Stats)
모든 결과값은 $S_{eff}$(유효 스탯) 기준입니다.

| 능력치 | 계산 공식 | 비고 |
| :--- | :--- | :--- |
| **최대 체력 (HP)** | $50 + (S1 \times 1.0) + (S2 \times 0.2) + (S3 \times 0.1)$ | 기본치 매우 낮음 (생존 지향) |
| **최대 마나 (MP)** | $30 + (S4 \times 0.4) + (S5 \times 0.1)$ | 장비 미착용 시 고위 스킬 제한 |
| **최대 스태미나** | $50 + (S6 \times 0.6) + (S7 \times 0.2)$ | 연속 공격/회피 제약 |
| **물리 공격력** | $10 + (S10 \times 0.15) + (S11 \times 0.1) + (S12 \times 0.05)$ | 상완(10) 영향력 최대 |
| **마법 공격력** | $10 + (S13 \times 0.2) + (S14 \times 0.1)$ | 에너지응집(13) 핵심 |
| **치명타 확률** | $5.0 + (S15 \times 0.1) + (S16 \times 0.1)$ | $30\%$ 초과 시 보정 계수 $0.1$ 적용 |
| **치명타 피해** | $150.0 + (S17 \times 0.1)$ | $200\%$ 초과 시 보정 계수 $0.1$ 적용 |
| **이동 속도** | $(S21 \times 0.05) + (S7 \times 0.03)$ | $20.0$ 초과 시 효율 $2\%$로 급감 |
| **명중/회피율** | $(S_{acc/dodge} \times Weight)$ 의 합 | 최대 $20.0\%$ 상한 (장비 가치 보존) |
| **상태이상 저항** | $5.0 + (S3 \times 0.03) + (S9 \times 0.02)$ | $20\%$ 초과 시 효율 $10\%$ 적용 |

### 2.3 던전 및 생활 시스템

| 항목 | 계산 공식 및 규칙 |
| :--- | :--- |
| **전투 행동 횟수**| $2 + ((S21 \times 0.4 + S22 \times 0.6 + S16 \times 0.2) / 15)$ (최소 2턴, 최대 5턴) |
| **화술 (할인율)** | $(S9 + S14) / 28.0$ (최대 90% 제한) |
| **비밀상자 할인** | $(S9 + S14) / 500.0$ (최대 80% 할인, 최소 100골드 보장) |
| **가챠 확률 가중치**| 행운(20)과 통찰(16) 스탯에 따라 전설/에픽 확률 가산 보정 |
| **노동 ST 소모** | $25 - ((S6 + S7) / 2)$ (최소 5 소모) |
| **탐사 효율** | 이동속도 보너스 + $(S23+S24) \times 0.01$ ($10$ 초과 시 효율 $10\%$ 적용) |
| **도망 확률** | $30\% + (S21 \times 0.1 + S24 \times 0.05)$ (최대 95%, 20% 초과 시 효율 $2\%$) |
| **휴식 회복량** | HP/ST: 기본 + ($S3 \times 5$), MP: 기본 + ($S8 \times 5$) |

## ⚔️ Combat Stats 필드 상세 정의 및 메커니즘

모든 비율(%) 스탯은 0.0 ~ 100.0 사이의 값을 기본으로 하며,
관통력(Pen)이나 저항력(Res)이 100을 초과할 경우의 예외 처리를 포함합니다.

---

### 1. 기본 생존 및 기동 (Survival & Utility)
| 필드명 | 타입 | 설명 | 수식 연동 |
| :--- | :---: | :--- | :--- |
| `maxHp` / `maxMp` | int | 생존 및 자원 최대치 | 기본값 + (스탯 보너스) |
| `maxStamina` | int | 행동 및 스킬 비용 | 기본값 + (스탯 보너스) |
| `hp/mpRegen` | double | 턴당 회복 고정 수치 | `Regen` 값 그대로 매 턴 가산 |
| `moveSpeed` | double | 탐사 효율 및 선공권 | `Exploration Efficiency`의 핵심 계수 |

---

### 2. 공격 및 적중 관련 (Offensive Stats)
| 필드명 | 타입 | 설명 (0~100%) | 비고 |
| :--- | :---: | :--- | :--- |
| `critRate` | % | 치명타가 발생할 확률 | 100% 초과 시 확정 치명타 |
| `critDmg` | % | 치명타 발생 시 피해 배율 | 기본 150.0% + `critDmg` |
| `accuracy` | % | 공격이 명중할 기본 확률 | 상대 `dodge`와 연산하여 최종 결정 |
| `dodge` | % | 적의 공격을 회피할 확률 | 100%일 때 완전 회피 (절대 명중 제외) |

---

### 3. 물리 및 마법 저항 메커니즘 (Defensive Core)
**공식:** `최종 대미지 = (Atk * (1 - (Res - Pen)/100)) - Def`

| 필드명 | 성격 | 작동 방식 | 특징 |
| :--- | :---: | :--- | :--- |
| `meleeAtk` | 고정값 | 물리 공격력의 기초값 | 무기 ATK와 합산 |
| `physDef` | 고정값 | 최종 대미지에서 뺄셈 | 후반부 다단 히트 잡몹 무력화용 |
| `physRes` | % | 물리 대미지 경감 비율 | 0 ~ 100% 사이로 작동 |
| `physPen` | % | 적의 `physRes`를 무시 | `Res`를 깎아 대미지 효율 증대 |

---

### 4. 상태 이상 시스템 (Status Ailments)
**판정:** `발생 확률 = (기본 부여율 + Pen) - StatusResist - 특정Res`

| 구분 | 저항(Res) / 관통(Pen) 필드 | 설명 |
| :--- | :--- | :--- |
| **공통** | `statusResist` | 모든 상태이상에 대한 범용 저항력 |
| **출혈** | `bleedRes` / `bleedPen` | `PhysRes`와 별개로 판정되는 지속 피해 |
| **군중제어** | `stunRes`, `frozenRes` | 턴 건너뛰기 등 행동 불능 관련 |
| **지속피해** | `burnRes`, `poisonRes` | 턴 종료 시 대미지 발생 |

---

### 5. 속성별 상세 스탯 (Elemental Stats)
각 속성은 `Atk(고정치)`, `Res(%)`, `Pen(%)`의 3요소로 구성됩니다.

| 속성계열 | 공격(Atk) | 저항(Res) | 관통(Pen) | 컨셉 및 특징 |
| :--- | :--- | :--- | :--- | :--- |
| **화염/빙결** | `fireAtk` | `fireRes` | `firePen` | 전형적인 마법 속성 상성 |
| **신성/암흑** | `holyAtk` | `holyRes` | `holyPen` | 종족(언데드 등)에 따른 상성 극명 |
| **독/카오스** | `toxicAtk` | `toxicRes` | `toxicPen` | 방어 무시 및 변칙 대미지 |

---

### 💡 관통(Pen)과 저항(Res)의 상호작용 규칙 (Edge Case)

1. **저항 100% 돌파 시:** - 적의 저항이 100%라면 대미지는 0이 됩니다. (흡수 로직 미구현 시)
    - 하지만 유저가 **관통(Pen)**을 30% 가지고 있다면, `100 - 30 = 70%`의 경감률만 적용되어 대미지가 들어갑니다.

2. **관통이 저항보다 클 때:**
    - 예: 적의 저항 10%, 나의 관통 40% -> `10 - 40 = -30%`.
    - 이 경우 저항력이 음수가 되어 **추가 대미지(130%)**가 들어가는 방식으로 설계 가능합니다. (Java 코드에서 이를 고려해 `Pen`을 더해주시면 됩니다.)

3. **반올림 규칙:**
    - 클래스 내 `roundOne` 메서드에 의해 모든 수치는 소수점 첫째 자리까지만 관리됩니다 (예: 15.5%).

### 🔍 정수 시스템 설계 가이드

1. **초반 부스팅 (9~7등급)**:
    - 1% 확률의 가치를 보존하기 위해 초반에는 수치를 과감하게 배정했습니다.
    - 9등급 정수 하나만으로도 초기 캐릭터는 사냥 속도가 비약적으로 상승합니다.

2. **후반 안정화 (4~1등급)**:
    - 1등급 정수 총합을 **330**으로 제한하여, 본체 성장의 가치가 묻히지 않도록 설계했습니다.
    - 1등급 정수(+330) + 레전드 장비 스탯 보너스(+100) + 본체 스탯(200~400)이 조합되어야 최종 꿈의 스탯인 **600** 지점에 도달할 수 있습니다.

3. **단일 스탯 투자 제한 (Soft Limit)**:
    - 정수 하나가 특정 스탯 하나를 100% 점유하는 것을 막아, 유저가 주력 스탯 외에도 명중, 회피, 저항 등 생존 스탯에 골고루 투자하도록 유도합니다.

4. **스킬과의 밸런스 (Trade-off)**:
    - 부여된 '스탯 총합' 외에도 각 정수 등급에 따른 **[전용 액티브 스킬]**이 부여됩니다.
    - 스탯 보너스가 기준치보다 낮을 경우, 그만큼 해당 정수의 스킬 위력이 강력하게 설계됩니다.

### 🛠️ 밸런스 체크 (1등급 기준)
- **공격력**: 정수 단일로 물리 공격력 관련 스탯에 투자 시 ATK 약 350~500 증가 (초기치 27의 약 15배).
- **치명타**: 정수 하나로 치명타 확률 소프트 캡(30%)을 즉시 졸업 가능.
- **방어력**: 정수 단일로 물리 방어 소프트 캡(200)의 약 85%를 충족 가능.

### 🔍 정수 시스템 설계 원칙

1. **스탯 보너스 분배 (Distribution)**
    - 부여된 '총합' 내에서 24개의 세부 스탯 ID에 자유롭게 분배됩니다.
    - **예시 (1등급, 총합 300)**:
        - [공격 올인형]: 물리 공격력 관련 ID(10, 11, 12)에 각 +100씩 분배 (O)
        - [올라운더형]: 모든 24개 스탯에 각 +12.5씩 분배 (O)

2. **단일 스탯 투자 제한 (Soft Limit)**
    - 정수 하나가 특정 스탯 하나를 지나치게 독점하는 것을 막기 위해 '단일 스탯 최대 투자' 제한을 둡니다. 나머지 수치는 다른 보조 스탯에 분배하여 캐릭터의 완성도를 높이도록 유도합니다.

3. **장비와의 시너지 (Synergy)**
    - 1등급 정수(+300)와 레전드 무기(+100 스탯 보너스)를 조합하면 본체 스탯 없이도 **+400**의 보너스를 얻습니다. 이는 본체 성장이 더해졌을 때 최종 도달점인 스탯 600 지점에 부드럽게 안착하게 합니다.

4. **가치 보존 (Rarity)**
    - 1% 드랍률의 가치는 '수치'뿐만 아니라 정수에 부여된 **[전용 액티브 스킬]**에서 나옵니다. 스탯 총합이 300으로 억제되어도, 스킬의 강력함이 등급의 차이를 완성합니다.

### 💡 도메인 적용 가이드 (Implementation Guide)

1. **ATK 합산 방식**: 몬스터의 `meleeAtk`가 100이고 `fireAtk`가 50이라면 총합 ATK 150으로 계산합니다. 하위 등급일수록 물리(`meleeAtk`) 비중이 높고, 상위 등급일수록 마법/속성 비중이 높아집니다.
2. **RES 적용 순서**: 대미지 계산 시 `physRes` 등 % 경감을 먼저 적용한 후, 남은 수치에서 `physDef` 고정치를 차감합니다.
3. **가변적 속성 배치**: 위 표의 RES 범위는 모든 저항의 평균값이 아니라, 해당 몬스터가 가질 수 있는 **최대 저항력의 상한선**입니다. (예: 9등급도 특정 속성 저항은 20%일 수 있으나, 평균은 5% 내외로 유지)

## 3. 핵심 밸런스 포인트 (Game Mechanics)

### ⚠️ 4층의 변수: 5등급 몬스터 (The 1% Encounter)
- **발생 확률:** 1% 미만 (매우 희귀)
- **시스템 연출:** 해당 몬스터 조우 시 화면 흔들림이나 전용 경고 로그 출력 권장.
- **공략 의도:** 현재 층 스펙으로는 절대 이길 수 없게 설정하여 '도망치기'나 '아이템 소모'를 유도하거나, 실력으로 잡았을 시 엄청난 스펙업 기회를 제공함.

### 🛡️ 성장 가이드라인
- **성장 정체 구간:** 6등급에서 5등급 진입 시 HP가 약 2.1배, ATK가 2.2배 점프합니다. 유저가 파밍의 필요성을 강하게 느끼는 첫 번째 구간입니다.
- **속도(SPD) 밸런스:** 고등급으로 갈수록 SPD를 높여 유저의 턴을 뺏거나 연속 행동을 하게 함으로써 수치 이상의 압박감을 줍니다.

### 🎁 보상 체계
- **드랍 테이블 존재:** 상위 등급으로 갈수록 드랍테이블이 존재하여 아이템, 부속물 등을 획득가능
